## 0. この回でやったことの全体像

1. **ログインできないバグの修正**（`index.html` の閉じタグのタイポ）
2. **jQueryアニメーション付きの開閉メニュー**を追加（ホーム ⇔ プライバシーポリシーの行き来）
3. **プライバシーポリシーページのヘッダー**をホームと同じ見た目に
4. **プライバシーポリシーページでもログアウトできる**ようにした

## 1. ファイルごとの役割（誰が何を担当しているか）

| ファイル | 役割 |
|---|---|
| `index.html` | ホーム画面。コーヒー記録の本体。Firebase一式 + 全JSを読み込む |
| `privacy-policy.html` | プライバシーポリシー画面。**最小限のFirebase**とログアウトだけ |
| `js/firebase-config.js` | Firebaseの**初期化設定**（どのプロジェクトに繋ぐか）。`firebase.initializeApp()` を実行 |
| `js/auth.js` | **ホーム画面専用**のログイン/新規登録/ログアウト/ログイン状態監視 |
| `js/api.js` | Firestore（データベース）の読み書き（保存・取得・更新・削除） |
| `js/app.js` | コーヒー記録のUI（カード描画・フォーム・グラフなど） |
| `js/menu.js` | ヘッダーの開閉メニューのアニメーション（jQuery）。**両ページ共通** |
| `js/modal.js` | 削除確認などのカスタムモーダル |
| `styles/main.css` | ホーム画面の見た目（配色の大元 `:root` もここ） |
| `styles/privacy-policy.css` | プライバシーポリシー画面の見た目 |

**ポイント：** `auth.js` と `api.js` と `app.js` は **3つでセット**。お互いの変数（`cardArea`, `coffeeLogs` など）や、ホーム画面のHTML要素（ログインフォーム等）に依存している。だから「ホーム画面以外」でそのまま読み込むと壊れる。

## 2. エージェントモードで書き換えた箇所（重点）

### 2-1.【最重要】ログインできないバグの修正

- **どこ**: `index.html` の `modal.js` 読み込みの閉じタグ
- **何が問題だったか**: `</scrip>` と書かれていた（`t` 抜け）
- **なぜ致命的か**: `<script>` は `</script>` 以外では閉じたと認識されない。そのため、本来その後ろにあった **Firebase本体（`firebase-app-compat.js`）の読み込みタグが「ただの文字」として飲み込まれ**、Firebaseが読み込まれなくなっていた。結果 `firebase` が存在せず、ログイン処理が全滅していた。
- **修正**: `</scrip>` → `</script>`

> 教訓：閉じタグ1文字のミスでも、HTML全体の読み込みが連鎖的に壊れることがある。

### 2-2. 開閉メニューの追加

- **追加したファイル**: `js/menu.js`（新規）
- **追加したHTML**: ヘッダー内の開閉ボックス
  ```html
  <div class="submenu">
      <button class="action-btn menu-btn" id="menu-box"> ≡ </button>
      <div class="submenu-content" id="submenu-content" style="display:none;">
          <a href="privacy-policy.html" class="submenu-item">プライバシーポリシー</a>
      </div>
  </div>
  ```
- **追加したCSS**: `.submenu` / `.submenu-content` / `.submenu-item`（`main.css` と `privacy-policy.css` の両方）
- **仕組み**:
  - `slideToggle()` … jQueryの関数。開いていれば閉じ、閉じていれば開く（高さアニメーション付き）
  - ページ遷移自体は普通の `<a href="...">` が担当。jQueryはあくまで「箱の開閉アニメーション」だけ
  - おまけで「メニューの外側クリックで閉じる」も実装

### 2-3. プライバシーページのヘッダーCSS

- **やったこと**: `main.css` を読み込むのではなく、必要なボタン/メニューのCSSを **`privacy-policy.css` に直接コピー**した
- **なぜ `main.css` を読み込まなかったのか（重要）**:
  - `main.css` の `body { margin: 0 }` が、`privacy-policy.css` の `body { margin: 0 auto }`（中央寄せ）を打ち消し、**全体が左寄り**になってしまうため
  - `auth.css` の `#logout-btn { display: none }`（ログイン後にJSで表示する設計）が効いて、**ログアウトボタンが消える**ため
  - → 競合を避けるため、プライバシーページは自分のCSSだけで完結させる方が安全

## 3.【最重要】ログアウト処理の理解

ここが一番ややこしいので詳しく。

### 3-1. ホーム画面（index.html）のログアウト

- 担当ファイル: `js/auth.js`
- 仕組み: `logoutBtn.addEventListener('click', () => firebase.auth().signOut())`
- さらに `onAuthStateChanged`（ログイン状態の監視）が、ログアウトを検知して自動的にログイン画面へ切り替える

### 3-2. プライバシー画面（privacy-policy.html）のログアウト

- **なぜ `auth.js` をそのまま使えないのか**:
  - `auth.js` は先頭で `auth-switch-link`（新規登録リンク）など **ホーム画面にしかない要素** を取得して `addEventListener` する
  - プライバシー画面にはその要素が無い → `null.addEventListener(...)` で**エラーになり、それ以降が全部止まる**
  - さらに `auth.js` は `app.js` の変数（`cardArea` 等）にも依存している
  - → だから「ホーム用の `auth.js` をそのまま読む」のはNG

- **代わりにやったこと**: プライバシー画面専用の小さなログアウト処理を直接書いた
  ```html
  <script>
      document.getElementById('logout-btn').addEventListener('click', function () {
          firebase.auth().signOut().then(function () {
              window.location.href = 'index.html';
          });
      });
  </script>
  ```
  - `firebase.auth().signOut()` … ログアウト命令
  - `.then(...)` … ログアウトが**完了したら**次の処理（＝ホームへ移動）
  - `window.location.href = 'index.html'` … ページをホームに切り替える

## 4. Firebase SDK の仕組みと、なぜ privacy-policy.html でも必要か

### 4-1. Firebase SDK は「部品」に分かれている

| 読み込むファイル | 役割 |
|---|---|
| `firebase-app-compat.js` | **土台**。`firebase` という入れ物と `initializeApp` を提供。これが無いと他も動かない |
| `firebase-auth-compat.js` | **認証**（ログイン・ログアウト）機能を追加 |
| `firebase-firestore-compat.js` | **データベース**（Firestore）の読み書き機能を追加 |

**SDK（エスディーケー）** … Software Development Kit。ある機能を使うための「道具一式」のこと。

### 4-2. なぜ privacy-policy.html でも Firebase が必要なのか

- プライバシー画面でも **ログアウト** をしたい
- ログアウト（`firebase.auth().signOut()`）には **認証機能**が要る
  - → `firebase-app-compat.js`（土台）+ `firebase-auth-compat.js`（認証）が必要
- 認証を使う前に「どのFirebaseプロジェクトに繋ぐか」を教える初期化が必要
  - → `firebase-config.js`（`firebase.initializeApp(...)`）が必要

### 4-3. なぜ Firestore は privacy-policy.html で「不要」なのか

- プライバシー画面では**データの読み書きをしない**（コーヒー記録の保存・取得はホームだけ）
- だから `firebase-firestore-compat.js` も `api.js` も読み込む必要がない
- 不要なものを読まない＝**軽く・壊れにくく**なる

### 4-4. privacy-policy.html が読み込むものの最終形

```
firebase-app-compat.js     … 土台（必要）
firebase-auth-compat.js    … 認証＝ログアウトに必要
firebase-config.js         … 初期化（必要）
menu.js                    … メニュー開閉（両ページ共通）
（インラインの）ログアウト処理
```
読み込まないもの：`firebase-firestore-compat.js` / `auth.js` / `api.js` / `app.js`

## 5. 覚えておくべき注意点

1. **`firebase-config.js` が2か所で使われる**
   - `index.html` と `privacy-policy.html` の両方が読み込む
   - Firebaseの設定を変えるときは、どちらのHTMLも確認する（configファイル自体は1つなので普通は触らなくてOK）

2. **キャッシュバスティング（`?v=N`）**
   - CSS/JSを変更してGitHub Pagesに上げるときは `index.html`・`privacy-policy.html` の `?v` の数字を上げる
   - 上げないと、スマホなどが古いファイルを表示し続ける

3. **「ホーム専用のJS」を別ページで読まない**
   - `auth.js` / `app.js` / `api.js` はホームのHTMLと一体。別ページで読むとエラーになる
   - 別ページで一部機能だけ欲しいときは、今回のように「必要な部品＋小さな専用処理」にする

---

## 6. 用語まとめ

- **SDK**: ある機能を使うための道具一式
- **初期化（initializeApp）**: 「どのプロジェクトに繋ぐか」をFirebaseに最初に教える処理
- **slideToggle（jQuery）**: 開閉をアニメーションで切り替える関数
- **signOut**: Firebaseのログアウト命令
- **`.then()`**: 非同期処理が終わったら次を実行する書き方
- **キャッシュバスティング**: ファイル名に `?v=N` を付けてブラウザに最新版を読ませる手法
